1. Introduction

The Online/Offline Faced Application is a Progressive Web App (PWA) developed for the DSWD Family Assistance Card in Emergencies and Disasters (FACED) program. It allows field enumerators to collect household data on a mobile device or laptop — online or offline — and sync records to the central database when connectivity is restored.

This manual covers all application modules and provides step-by-step instructions for encoding (data entry), verifying beneficiaries, generating reports, and administering the system.

1.1 Who Uses This App?

RolePrimary ModuleDescription
EnumeratorEncodingRegisters affected families using the FACED form on a device in the field.
VerifierVerifyingChecks for duplicate registrations before or during encoding.
AdministratorAdmin, RecordsManages access codes, evacuation centers, synced records, and printing.
Coordinator / DSWD StaffDashboardViews aggregated reports on evacuation centers, demographics, and shelter damage.

1.2 Geographic Scope

The app is configured for Sarangani Province, Region XII. Municipality and barangay dropdowns are pre-populated with Sarangani locations.

2. Getting Started

2.1 Opening the App

  1. Open the app URL in a modern web browser (Chrome, Edge, or Firefox recommended).
  2. For best offline experience, install the PWA: use your browser's "Install app" or "Add to Home Screen" option.
  3. The app works on phones, tablets, and desktop computers.

4. Encoding Module — Data Entry Guide

ENCODING

In this app, "encoding" means entering or registering a FACED household record — not text/character encoding. This is the primary task for field enumerators.

4.1 Logging In as an Enumerator

  1. Go to the Encoding module (home page).
  2. Enter your access code in the format FACED-XXXX-XXXX.
  3. You must be online for first-time login.
  4. Each access code works on one device only. Do not share your code with another enumerator on a different device.
  5. Tap Continue to FACED form.
Important: If you are offline and have never logged in on this device, login will not work. Connect to the internet first.

4.2 Downloading the Offline Data Bundle

On first login (while online), the app automatically downloads:

  • Previously synced family records — for offline duplicate checking
  • Evacuation center library — for EC site suggestions in the form

Wait for the download to complete before starting encoding. A progress modal will appear. Once finished, you can encode offline.

4.3 Filling Out the FACED Form

Address line, city/municipality, barangay, province, and zip code.

Section E — Other Information

  • Indigenous Peoples (IP) ethnicity
  • Religion

Section F — Family Information (Item 25) — Required

Add at least one household member with name and relationship. For each member, enter:

  • Name, relationship to head, birthdate, age, sex
  • Education, occupation
  • Vulnerability flags: Pregnant, Lactating, Elderly, PWD, Infant (Elderly/Infant auto-set from age)

Tap Add member to add more household members.

Section J — Date Registered & Data Privacy

  • Date registered — defaults to today
  • Data Privacy Declaration — you must check the consent box (RA 10173) before saving

4.4 Duplicate Checking During Encoding

When you enter the head of family's first and last name, the app automatically checks for possible duplicates in three sources:

  1. Online database — if connected to the internet
  2. Offline cache — previously downloaded synced records
  3. This device — records saved locally on your device

If a possible duplicate is found:

  • An alert appears showing where the match was found
  • A dialog offers: Go back, Continue encoding, or Save anyway
  • You must acknowledge the duplicate before saving
Tip: Always verify details with the household even when no duplicate is found.

4.5 Saving a Record

  1. Complete all required fields and acknowledge the privacy declaration.
  2. Tap Save FACED record.
  3. A dialog shows the Family Serial Number assigned to the record.
  4. Tap Copy serial to copy the number for printing or verification.
  5. The record is saved locally on your device with status Pending (unsynced).

4.6 Managing Saved Records

Below the form, the Saved records section lists all records on your device.

4.7 Complete Encoding Workflow Summary

  1. 1 Receive access code from administrator
  2. 2 Log in online and wait for offline bundle download
  3. 3 Go to the field — encode households (online or offline)
  4. 4 Save each record; note the family serial number
  5. 5 Return to connectivity and tap Sync online
  6. 6 Optionally export Excel backup locally

5. Verifying Module

VERIFYING

The Verifying module lets verification staff check whether a beneficiary is already registered before encoding proceeds.

5.1 Logging In

  1. Go to Verifying in the navigation bar.
  2. Enter the verify password provided by your administrator.

5.2 Duplicate Check Tab

  1. Enter the beneficiary's first name and last name (required).
  2. Optionally add middle name, birthdate, municipality, and barangay to narrow results.
  3. Tap Search.

Results:

  • No match found — "You may proceed with encoding." Always confirm details with the household.
  • Match found — Review cards showing encoder name, registration date, and location. Investigate before proceeding.

5.3 Export Excel Tab

Download synced online records filtered by municipality and barangay as an Excel file.

5.4 Offline Data Tab

Download a full copy of synced records to your device for offline duplicate checking:

  1. Connect to the internet.
  2. Tap Download on the Offline Data tab.
  3. Wait for download to complete — record count and timestamp are shown.
  4. Use Clear cache to remove the offline copy if needed.

7. Admin Module

ADMIN

Administrators manage access codes, monitor encoding activity, and maintain the evacuation center library.

7.1 Logging In

Go to Admin and enter the admin password.

7.2 Summary Tab

  • Per-enumerator statistics: encoded count, codes active/used/rejected
  • Daily encoding chart — records synced per day (7, 14, or 30-day view)
  • Duplicate and trash metrics
  • Export Excel per enumerator

7.3 Create Codes Tab

  • Batch generate — create 1 to 100 codes at once
  • Custom code — add a specific code with assignee name and email

Codes follow the format FACED-XXXX-XXXX.

7.4 Manage Codes Tab

ActionDescription
SearchFind codes by value or assignee
Assign enumeratorSet name and email for a code before distribution
Reject codeDeactivate a code so it cannot be used
Export used codesDownload Excel of all used codes

Code lifecycle: Active → Used (bound to one device) or Rejected

7.5 EC Library Tab

Maintain the list of evacuation center sites used in the encoding form:

  1. Select municipality and barangay.
  2. Enter the evacuation center name.
  3. Tap Add.
  4. Use Delete to remove outdated sites.

Changes appear in encoding form suggestions after enumerators refresh their offline bundle.

9. Printing FACED Forms & ID Cards

Printing is accessed from the Records module — not from the main navigation.

9.1 Print FACED Annex

  1. In Records, find the record and tap Print FACED.
  2. A new tab opens with the formatted annex form.
  3. The browser print dialog opens automatically.
  4. Select your printer and print.

9.2 Generate FACED ID

  1. In Records, tap Generate FACED ID on a record row.
  2. A new tab opens with the ID card layout including a QR code.
  3. The print dialog opens automatically.
Pop-up blocker: If nothing opens, allow pop-ups for this site in your browser settings, then try again.

10. Offline Use & Syncing

10.1 How Offline Storage Works

Records you save during encoding are stored in your device's local database (IndexedDB). They remain on your device until you sync them to the server.

10.2 Sync Status Labels

StatusMeaning
PendingSaved locally, not yet uploaded
SyncedSuccessfully uploaded to the server
FailedUpload attempted but failed — retry when online

10.3 Syncing Records

  1. Connect to the internet.
  2. Go to Encoding → Saved records.
  3. Tap Sync online.
  4. Wait for confirmation: "Successfully synced N record(s) online."

10.4 Refreshing Offline Data

If you need updated duplicate-check data or EC library entries, log out and log back in while online to trigger a fresh offline bundle download.

11. Troubleshooting

ProblemSolution
Cannot log in — "offline" message Connect to the internet. First-time login requires an active connection.
Access code rejected Code may be used on another device, rejected by admin, or typed incorrectly. Contact your administrator.
"Encoder session expired" Log out and log back in with your access code while online.
Sync fails or shows "Failed" Check internet connection. Retry Sync online. If problem persists, export Excel backup and contact admin.
Duplicate check stuck on "Checking…" Check connectivity. Offline mode uses cached data only — wait or proceed with caution.
Print / ID generation blocked Allow pop-ups for this site in browser settings.
EC site not in suggestions Ask administrator to add the site in Admin → EC Library, then re-download offline bundle.
Form won't save Check all required fields, privacy checkbox, evacuation center selection, and family member entries.
